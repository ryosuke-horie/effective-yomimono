# ブックマーク自動分類機能 - ユーザーストーリー

## 概要

ブックマーク自動分類機能は、ユーザーが保存した技術記事を AI が自動的に分析し、適切なカテゴリに分類・整理するものです。この機能により、ユーザーは手動での整理作業から解放され、保存した記事を効率的に管理できるようになります。

## ユーザーストーリー

### ストーリー 1: 新しく追加したブックマークの自動分類

```
身近な技術ブログからニュースサイトまで、日々の仕事や学習に役立ちそうな記事をいくつも保存している開発者のケイちゃん。
ブラウザでは一度に5〜10個のタブを開いていることが多く、それらをまとめて Effective Yomimono に保存しています。
ただ、保存した記事が増えるにつれ、何がどのカテゴリに属するのか把握しにくくなってきていました。

Effective Yomimono の自動分類機能を試すことにしました。
10件ほど未分類のブックマークがあったので、「最近保存した記事を自動で分類してほしい」と指示します。

AIアシスタントは「過去1週間に保存された未分類のブックマークが10件あります。自動分類を実行しますか？」と確認してきます。
「はい、お願いします」と応答すると、AIは記事の内容を解析し始めました。

待つこと数秒、AIアシスタントは「分類が完了しました」と報告し、以下のような分類結果を表示しました：

- フロントエンド開発: 4件
  - React: 2件（「Reactのレンダリング最適化戦略」「React Server Componentsの導入と実践」）
  - CSS: 2件（「モダンなCSS変数の使い方」「CSSコンテナクエリ実践ガイド」）
  
- バックエンド開発: 3件
  - Node.js: 1件（「Node.js 20の新機能と改善点」）
  - データベース: 2件（「SQLiteの高度な使い方」「NoSQLとSQLのハイブリッド戦略」）
  
- クラウド技術: 3件
  - Cloudflare: 2件（「Cloudflare Workersでエッジコンピューティング」「D1データベースの実用的なパターン」）
  - AWS: 1件（「Lambda関数の効率的な管理方法」）

AIはさらに「これらの分類を適用しますか？必要に応じて調整も可能です」と提案してきます。
ケイちゃんは「Cloudflareの2つは『サーバーレス』というカテゴリにまとめたい」と指示しました。

AIは「了解しました。変更を適用します」と応答し、再構成した分類を適用しました。
その結果、フロントエンド関連、バックエンド関連、サーバーレス関連とカテゴリ分けされたブックマークが、整理されたビューで表示されるようになり、後で調べたい技術や記事を見つけやすくなりました。
```

### ストーリー 2: 既存ブックマークの再分類とカテゴリ整理

```
数ヶ月間 Effective Yomimono を利用してきた結果、100以上の技術記事をブックマークしたエンジニアのミケちゃん。
最初は手動でカテゴリ設定していましたが、途中で面倒になり多くが未分類のままになっていました。

ブックマークを整理したいと思い、「すべての未分類ブックマークを自動で整理してほしい」とAIアシスタントに依頼します。

AIアシスタントは「未分類のブックマークが65件見つかりました。大量のブックマークを処理するため、分類の粒度を選択してください：1. 大まかなカテゴリのみ 2. 詳細なサブカテゴリまで」と質問してきます。

「詳細なサブカテゴリまでお願いします」と返答すると、AIは分析を開始しました。

処理が完了すると「65件のブックマークを7つの主要カテゴリと23のサブカテゴリに分類しました」という報告とともに、分類結果のサマリーが表示されます：

1. フロントエンド開発 (23件)
   - JavaScript/TypeScript (8件)
   - React/Next.js (10件)
   - CSS/デザイン (5件)

2. バックエンド開発 (18件)
   - Node.js (6件)
   - API設計 (4件)
   - データベース (8件)

3. DevOps (7件)
   - CI/CD (3件)
   - コンテナ (4件)

4. クラウド技術 (9件)
   - サーバーレス (5件)
   - クラウドアーキテクチャ (4件)

5. パフォーマンス最適化 (3件)

6. セキュリティ (2件)

7. 技術トレンド (3件)

AIは「この分類でよろしいですか？修正が必要な箇所はありますか？」と確認してきます。

ミケちゃんは「パフォーマンス最適化とセキュリティは数が少ないから『開発プラクティス』というカテゴリにまとめてほしい」と指示しました。

AIは指示に従って再構成し、「変更を適用しました。他に調整が必要な点はありますか？」と尋ねてきます。

「これで大丈夫です。ありがとう！」と応答するとAIはブックマークにカテゴリとサブカテゴリのメタデータを適用し、整理されたブックマーク一覧が表示されるようになりました。

後日、ミケちゃんが「フロントエンド関連の記事だけ表示して」と指示すると、AIはすぐにフロントエンド開発カテゴリのブックマークリストを表示し、知識の探索がずっと容易になりました。
```

### ストーリー 3: 継続的な自動分類

```
毎日多くの技術記事を読み、気になるものを Effective Yomimono に保存している技術リードのトラちゃん。
Chrome拡張機能を使って、まとめて記事を保存することが多いのですが、分類は自動的に行われてほしいと考えています。

トラちゃんは設定画面で「新しいブックマークを自動的に分類する」オプションを有効にしました。

翌日、5つの記事をまとめて保存した後、Effective Yomimonoを開くと、AIアシスタントから
「5件の新しいブックマークを以下のカテゴリに自動分類しました」という通知と分類結果が表示されます：

- クラウドネイティブ (2件)
  - 「KubernetesのOperatorパターン実践ガイド」
  - 「マイクロサービスにおけるサーキットブレーカーの実装」

- 機械学習 (1件)
  - 「生成AIアプリケーションの実装パターン」

- アーキテクチャ (2件)
  - 「イベント駆動アーキテクチャの設計原則」
  - 「ドメイン駆動設計におけるコンテキストマッピング」

トラちゃんは分類結果を確認し、一部修正したいと思い「アーキテクチャの2つの記事は『システム設計』というカテゴリにしてほしい」と指示します。

AIは「カテゴリ名を『システム設計』に変更しました」と応答し、変更を反映しました。

数週間後、トラちゃんはブックマークの分類パターンをAIが学習していることに気づきます。
トラちゃんが頻繁に「システム設計」を「アーキテクチャパターン」に変更していたため、
AIがそのパターンを学習し「今回保存された『ヘキサゴナルアーキテクチャの実装例』は『アーキテクチャパターン』カテゴリに分類しました」と表示するようになっていました。

この継続的な学習と自動分類により、トラちゃんのブックマーク管理の負担が大幅に軽減され、
技術情報の整理と再利用がより効率的になりました。
```
